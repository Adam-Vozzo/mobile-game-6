# Android build & signing — Project Void

How to produce a runnable APK on the test device (Nothing Phone 4(a) Pro)
and, eventually, a signed bundle for Play Store distribution. **No keystore
is generated by this repo. The keystore is generated and held by the human
on a local machine and never committed.**

---

## One-time setup (per developer machine)

1. **Install the Godot editor** matching the version in `project.godot`'s
   `config/features` (currently 4.6, Mobile build).
2. **Install the Android Editor support templates** via Editor → Manage Export
   Templates → Download for the same version.
3. **Install the Android SDK & build tools.** Easiest path is Android Studio:
   - SDK Platform: API 34 (target).
   - SDK Build-Tools: 34.x.
   - SDK Platform-Tools (for `adb`).
   - NDK: only required if `gradle_build/use_gradle_build=true` and we
     custom-build the template. Skip for now.
4. **Install a JDK 17.** Required by the Android Editor and gradle build.
5. **Configure Godot Editor → Editor Settings → Export → Android** with:
   - `Java SDK Path` → JDK 17 install root.
   - `Android SDK Path` → the SDK root from step 3.
   - `Debug Keystore` → leave default (Godot's debug keystore) for editor
     `One-Click Deploy` while developing.
6. **Enable USB debugging** on the test device (Settings → System →
   Developer options → USB debugging). Plug in over USB and accept the
   debug-key prompt on the device.

Verify with `adb devices` — you should see the device serial listed.

## Daily build (debug, on-device)

From the editor:

1. Connect the test device via USB. `adb devices` should list it.
2. In Godot, top-right "One-Click Deploy → <device>" → Run.
3. Editor will export, install, and launch the debug APK.

From the CLI (Linux/macOS):

```sh
# Editor headless export to debug APK in build/
godot --headless --export-debug "Android" build/project_void_debug.apk

# Push to device
adb install -r build/project_void_debug.apk

# Launch via the package's default LAUNCHER intent (works regardless of
# what Godot names the activity in this version):
adb shell monkey -p com.projectvoid.stray \
    -c android.intent.category.LAUNCHER 1
```

If you prefer the explicit-activity form, run `adb shell dumpsys package
com.projectvoid.stray | grep -A1 LAUNCHER` after install to find the
current activity name and pass it to `am start -n <package>/<activity>`.

## Release / Play Store build (deferred — needs human)

This is in the **What requires the human** list. Don't run it without a
green light. Steps are listed for reference only.

1. **Generate a keystore** (one time, kept off-repo):

   ```sh
   keytool -genkey -v -keystore project_void_release.keystore \
       -alias projectvoid -keyalg RSA -keysize 2048 -validity 25000
   ```

   Store this file outside the repo (e.g. `~/.keys/`). Keep the keystore
   and alias passwords in a secure location. **Do not commit the keystore
   or the passwords.** `~/.keys/` is the convention in this repo's docs.

2. **Configure the release preset** (Editor → Project → Export → Android):
   - `Encryption` → off unless we add encrypted assets.
   - `Keystore → Release` → path to the keystore (absolute or env var).
   - `Keystore → Release User` → alias (`projectvoid`).
   - `Keystore → Release Password` → keep blank in the file; supply at
     export time or via the Godot editor settings (so the password never
     touches version control).
   - Flip `gradle_build/use_gradle_build` to `true` (custom build path).

3. **Bump version** in `export_presets.cfg`:
   - `version/code` increments by 1 each Play Store upload.
   - `version/name` follows semver.

4. **Export AAB** (Android App Bundle, required for Play Store):
   - Set `gradle_build/export_format=1` (AAB).
   - Output file: `build/project_void_<version>.aab`.

5. **Upload to Play Console** under the Internal Testing track first.

## First-run verification (current Gate 0 state)

After import + one-click deploy of `scenes/levels/feel_lab.tscn`:

In editor:

- [ ] Project opens in Godot 4.6 with **no errors** in the Output panel.
      Warnings about Forward+ vs. Mobile are OK; missing autoload scripts
      are not — fix paths if any appear.
- [ ] F5 → Feel Lab loads. You should see a grey concrete arena with
      platforms rising +X, slopes at -X, walls at -Z, a moving platform
      ping-ponging at +Z, and four tall pillars on the corners under
      heavy fog.
- [ ] **WASD / arrow keys** move the Stray. **Space** jumps. Holding
      jump gives a higher arc; tapping cuts the apex (variable jump).
- [ ] Walk off the level → the Stray falls below `fall_kill_y` (-25),
      plays the placeholder reboot effect (red flash → dark frame →
      warm power-on), and respawns at the spawn marker.
- [ ] **R** respawns at any time.
- [ ] **F1** opens the dev menu. Profile dropdown shows Snappy. Four
      sliders adjust live. The perf line at the bottom updates.
- [ ] Right-mouse drag rotates the camera. Releasing and standing still
      eventually auto-recenters behind the player's last movement
      direction (~1.2 s idle).

On device (after `adb install`):

- [ ] Game launches in **landscape** (sensor-landscape — both directions
      are accepted, portrait is not).
- [ ] **Left half of the screen** = virtual stick. Touch and drag.
      Origin appears at first contact (free-floating).
- [ ] **Right side, bottom-right** = jump button (red translucent
      circle). Tap to jump, hold for higher.
- [ ] **Right half of the screen** outside the jump button = camera drag.
      Drag to rotate the camera; release and idle to auto-recenter.
- [ ] **3-finger tap** opens the dev menu.

## Performance verification on device

Each iteration that touches gameplay should run on device and capture:

- Frametime (ms) at the worst point in the Feel Lab.
- Triangles in frame and draw calls (from `PerfBudget` autoload, surfaced
  in the dev menu's perf overlay).
- Battery temperature after a 5-minute run (thermal throttling is the
  reason we budget 8–10 ms instead of 16.6).

Log results in the iteration's README entry. Regressions ≥ 0.5 ms or ≥ 10%
in draw calls block merging without a written reason in `DECISIONS.md`.

## Troubleshooting

- **`One-Click Deploy` greyed out**: editor can't see `adb`. Re-check the
  Android SDK Path in Editor Settings, then restart the editor.
- **Black screen, then crash**: usually missing Android Editor templates
  for the matching Godot version. Re-download from the Manage Export
  Templates dialog.
- **Touch input not received**: verify `display/window/handheld/orientation`
  is sensor-landscape (4) in `project.godot`, not portrait, and that
  `pointing/emulate_touch_from_mouse` is on (it should be — we want the
  same input layer in editor and on device).
- **Frame drops only on device, not in editor**: expected. Mobile renderer
  has different cost shapes than Forward+. Test on device early and often.
